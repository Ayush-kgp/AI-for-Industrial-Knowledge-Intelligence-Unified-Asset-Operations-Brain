# Industrial Knowledge Intelligence: Unified Asset & Operations Brain

An AI platform that turns fragmented industrial documents (engineering drawings, maintenance records, SOPs, inspection reports) into a single, citation-grounded, graph-aware knowledge base. By unifying unstructured and structured data, this platform empowers engineers to **discover** knowledge instantly instead of **searching** for it.

Built for the **AI for Industrial Knowledge Intelligence** hackathon track.

## The Problem

Professionals in asset-intensive industries spend an average of **35% of working hours** searching for information or recreating documents that already exist. A typical large plant operates across 7–12 disconnected document systems. This fragmentation contributes to nearly a fifth of all unplanned downtime. 

Traditional RAG (Retrieval-Augmented Generation) systems fail here because they lack the relational context required to traverse complex, multi-hop operational queries (e.g., "What maintenance followed the Q3 pressure trip?").

## The Solution

We implemented an **Agentic Knowledge Graph architecture**:
1. **Universal Document Ingestion**: Parses PDFs and unstructured logs.
2. **Knowledge Graph (Neo4j)**: Maps structural relationships between equipment, documents, people, and events.
3. **Semantic Search (Qdrant)**: Stores deep chunk embeddings for textual relevance.
4. **Agentic LLM Planner**: A multi-step reasoning agent (OpenAI function-calling) that dynamically chains tool calls across the graph and vector databases to synthesize complex answers.

### Core Guiding Principle: Trust
Every single claim generated by the Agentic Planner is strictly grounded. The system is constrained by an anti-hallucination guardrail that forces all output to include explicit **citations and snippets** from the source documents.

## Repository Architecture

- `/app/pipeline/`: Ingestion scripts (parsing, chunking, graph extraction, and metadata writing).
- `/app/retrieval/`: The Agentic Planner and database query tools.
- `/app/main.py`: FastAPI backend exposing the planner via REST.
- `/frontend/`: Next.js + Tailwind web application with a live Chat & Citations UI.
- `ARCHITECTURE.md` & `SCHEMA.md`: Deep dives into system design and database schemas.

## How to Run Locally

### 1. Start Databases
Run the provided `docker-compose.yml` to spin up PostgreSQL, Neo4j, and Qdrant locally.
```bash
docker-compose up -d
```

### 2. Start the Backend API
```bash
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 3. Start the Frontend UI
```bash
cd frontend
npm install
npm run dev
```

Visit `http://localhost:3000` to interact with the Copilot!
